대학원생으로서의 분석적 전문성을 유지하면서도, 별도의 비용이나 서버 구축 없이 로컬 환경에서 즉시 실행 가능한 MVP(Minimum Viable Product) 형태의 PRD로 수정해 드립니다.

## 프로젝트 PRD: 로컬 환경 기반의 사용자 행동 데이터 분석 및 실험 설계 MVP

### 1. 프로젝트 개요 (Overview)

본 프로젝트는 데이터 분석가로서 비즈니스 문제를 정의하고, 가상의 로그 데이터를 생성하여 통계적 가설 검정 및 리텐션 분석을 수행하는 전체 과정을 로컬 환경에서 구현합니다. 대학원 수준의 분석 방법론을 실무적인 대시보드 형태로 시각화하여 데이터 분석가의 핵심 역량을 증명하는 것을 목표로 합니다.

### 2. 문제 정의 및 가설 (Problem & Hypothesis)

* 비즈니스 문제: 특정 건강 관리 앱에서 유저의 초기 이탈률이 높고, 보상 획득 경험이 리텐션에 미치는 영향이 불분명함.
* 분석 가설: 가입 후 24시간 이내에 보상을 1회 이상 획득한 유저는 그렇지 않은 유저보다 7일 차 리텐션이 유의미하게 높을 것이다.
* 실험 가설: 보상 획득을 독려하는 개인화 푸시 알림을 발송할 경우, 대조군 대비 보상 전환율이 10% 이상 상승할 것이다.

### 3. 로컬 MVP 기술 요구사항 (Technical Requirements)

비용 발생을 방지하고 유지보수 부담을 최소화하기 위해 아래 도구들만 사용합니다.

* 데이터 저장소: SQLite (별도 설치 없는 파일 기반 DB)
* 분석 엔진: Python (Pandas, Numpy, Scipy, Statsmodels)
* 시각화 및 배포: Streamlit (로컬 웹 대시보드)
* 버전 관리: GitHub (분석 코드 및 결과물 관리)

### 4. 핵심 기능 및 분석 범위 (Core Features)

| 구분 | 주요 내용 | 비고 |
| --- | --- | --- |
| 데이터 생성 | Python을 활용한 유저 로그(가입, 행동, 실험) 합성 데이터 생성 | 대학원 수준의 분포 설계 적용 |
| 리텐션 분석 | SQL 쿼리 기반의 코호트 분석 및 리텐션 커브 시각화 | SQLite 활용 |
| 실험 검정 | A/B 테스트 결과의 통계적 유의성 검정 (P-value, Effect Size) | Scipy/Statsmodels 활용 |
| 세그먼트 분석 | 행동 패턴 기반의 유저 클러스터링 및 그룹별 특징 추출 | Scikit-learn 활용 |
| 통합 대시보드 | 위 모든 결과를 한눈에 확인 가능한 로컬 웹 페이지 | Streamlit 활용 |

### 5. 데이터 스키마 (Local Data Schema)

로컬 SQLite DB에 저장될 최소한의 테이블 구조입니다.

* users: user_id, signup_date, channel, segment
* user_logs: log_id, user_id, event_name, event_timestamp, value
* ab_test_results: user_id, group_name(A/B), is_converted, conversion_timestamp

### 6. 분석 방법론 (Methodology)

대학원생의 강점을 보여줄 수 있는 분석적 깊이를 포함합니다.

* 단순 평균 비교를 넘어, 교란 변수를 통제하기 위한 성향 점수 매칭(PSM) 등 인과추론 기법 적용 가능성 검토.
* 실험 결과 해석 시 신뢰 구간(Confidence Interval)과 통계적 검정력(Power) 분석 포함.
* 유저 세그먼트별로 실험 효과가 다르게 나타나는지 확인하는 이질적 처치 효과(HTE) 분석.

### 7. 성공 지표 (Success Metrics)

* Primary: 7일 차 리텐션 및 실험군/대조군 간의 전환율 차이.
* Technical: 데이터 생성부터 시각화까지의 파이프라인 자동화 여부.
* Product: 분석 인사이트로부터 도출된 구체적인 비즈니스 액션 아이템 3가지 이상.
